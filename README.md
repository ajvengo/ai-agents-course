# Урок «LLM и AI-агенты» — Quarto-проект

Proof-of-concept: тот же иллюстрированный урок, но как **docs-as-code**.
Исходник — Markdown в git, итог — HTML и PDF, собранные `quarto render`.

## Структура

```
_quarto.yml              конфиг проекта: форматы html + pdf, тема, ассеты
llm-i-agenty.qmd         сам урок — Markdown + fenced divs (это и есть исходник)
theme/lesson.scss        тёплая редакторская тема (Quarto SCSS: defaults + rules)
assets/lesson-foot.html  JS: вкладки с кодом + интерактивный квиз
_diagrams/*.qmd          5 диаграмм-партиалов, инлайнятся через {{< include >}}
_diagrams/*.png          растровые версии диаграмм — подставляются в PDF
.github/workflows/       CI: quarto render → GitHub Pages (HTML) + PDF-артефакт
```

## Сборка

Нужен [Quarto](https://quarto.org) 1.4+ (для PDF — ещё LaTeX: `quarto install tinytex`).

```sh
quarto preview                 # живой просмотр HTML с hot-reload
quarto render                  # собрать всё (html + pdf) в _site/
quarto render --to html        # только HTML
quarto render --to pdf         # только PDF
```

## Что показывает PoC

- **Исходник диффабельный.** `llm-i-agenty.qmd` — почти чистый Markdown;
  правка текста урока = правка одной строки, видная в `git diff`.
- **Один источник → два формата.** `quarto render` собирает HTML и PDF из
  одного `.qmd`. Форматы настраиваются в `_quarto.yml`.
- **Дизайн — это тема.** Вся вёрстка вынесена в `theme/lesson.scss`;
  контент о ней ничего не знает. Новый урок переиспользует ту же тему.
- **Диаграммы — отдельные файлы.** Каждая SVG лежит в `_diagrams/` и
  диффится отдельно от текста.

## Честные оговорки

- **HTML — основной формат.** Полный дизайн, интерактивные вкладки и
  квиз со счётом.
- **PDF — типографский / архивный вид.** Интерактив уплощается: вкладки
  с кодом разворачиваются в последовательные блоки, квиз становится
  статичным «вопрос + варианты + разбор». Диаграммы автоматически
  переключаются с инлайн-SVG на PNG (через `content-visible`).
  Для SVG-конвертации в LaTeX-пути нужен `librsvg` — здесь обойдён
  PNG-фолбэком, так что лишние зависимости не требуются.
- **Компоненты — кастомные fenced divs** (`.callout-box`, `.quiz`,
  `.code-stages` …), стилизованные темой. Можно вместо них взять
  нативные Quarto-callouts (`::: {.callout-note}`) и `panel-tabset` —
  тогда PDF-деградация будет «из коробки», но контроль над дизайном
  меньше. Это осознанный размен в пользу пиксельной точности.
- **Навигация** — нативный Quarto TOC (слева) вместо кастомного
  sticky-меню оригинала.
